关键发现

1

PGC 投入足额，但 UGC 几乎带不起

元宝 PGC 5.2 万（行业第 2）但 UGC/PGC 比例和仅为 0.13x（行业垫底，豆包 7.14x）

  • 产品用户画像偏 B 端 / 职场 —— 这类用户分享欲低于 C 端娱乐用户；
  • 缺失情感钩子 —— 专业工具类内容不易引发二创；
  • 阵地选择偏微信公众号（腾讯生态内）—— 微信转发传播效率远低于抖音/小红书。
3

高讨论意愿的场景特点

  • 强情绪宣泄——用户使用 AI 进行情感倾诉或角色扮演；
  • 强结果展示——AI 生图、文、视频，晒出自己的产出，并激发模仿/讨论；
  • 强反常识——玩梗（搞笑反差），颠覆性技术；
  • 强场景代入——同人 / 角色 / 故事激发想象力。

元宝目前主要推出专业工具类内容不易引发二创，应从创意写作（学术故事性）、课业辅导（学习成果可炫耀）、职场工具与职场陪伴角度切入，角色扮演市场借助腾讯内容生态（王者、QQ）进行差异化，激发用户情感共鸣。

4

阵地优化策略

  • 微信公众号：元宝需要选择或培养能够持续产爆款的"个人 IP 型"KOL，采取真实使用故事分享；
  • 小红书：有量无讨论，必须推出更多真实种草内容，如在具体的应用场景下介绍元宝的功能；
  • 微博：有量无讨论，需借助微博本身社会热点、文娱热点的讨论度宣传元宝。
5

元宝两大机会点

  • 角色扮演：市场由 ds 和豆包主导，但是用户提到元宝"人设细节稳"这一特点，可作为差异化方向；
  • 会议纪要：用户对元宝的满意点主要体现使用腾讯会议的时候直接开启元宝会议纪要，提高效率。

3.2行业投放情况

主推功能错位

全行业都在主推 Agent 功能，投入普遍过剩（占整体 PGC 的 10%-40%），但用户普遍不会跟风讨论，各产品 Agent 功能在 UGC 中的占比均低于 10%，Agent 营销叙事需要重新设计，元宝暂时不推荐抢行追随市场大力推广 Agent 功能；

3.3元宝策略

  • 行业动态 DeepSeek 已主导（市场率 65.5%），元宝跟推无效，但元宝"节点式"的真新闻发布反而引发了真实 UGC 讨论，腾讯混元 Hy3 preview 发布当日和次日引发 UGC 讨论（发布前 UGC 每日 2-5 条，发布当日和次日 UGC 分别为 42、75 条），因此建议从追赶式报道改为节点式报道（如，混元发布）。
